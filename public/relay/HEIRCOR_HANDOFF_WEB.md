## SECTION 10 — THE NEW DIRECTION: HEIRCOR OP. WEB PORTAL

### Dan's Vision
An **"Ouch Wikipedia"** hosted online, themed as the **in-world Heircor Op. online portal** — as if Heircor Op. has built a classified intelligence database about the Ouch universe. Users browse it like a wiki but it feels like accessing a corporate espionage intranet.

### Goals
- Host the Epictome content as navigable web articles
- Expandable: Dan can add new intel drops, terminology, synchronicities
- Shareable: friends/collaborators can browse
- Controllable: Dan maintains the source of truth
- Potentially: live updates as the project evolves (new albums, new issues, etc.)

### Recommended Tech Stack

**Option A — Simple & Deployable Now (Recommended Start)**
- **Astro** (static site generator) + vanilla CSS
- Markdown files per article (easy to edit, version-controllable)
- Hosted on **Vercel** or **Netlify** (free tier, deploys from GitHub)
- GitHub repo = source of truth + version history
- No database needed; flat file architecture

**Option B — Wikipedia-Like with CMS**
- **Next.js** + **Sanity.io** (headless CMS) or **Notion API**
- More complex but gives Dan a visual editor for updates
- Still deployed via Vercel

**Option C — Maximum Control / Local First**
- Pure HTML/CSS/JS single-page app with JSON data files
- Self-contained; can run offline; host anywhere
- Best for: "I want to control everything and understand all of it"

### Recommended Architecture for the Portal

```
heircor-portal/
├── public/
│   └── assets/ (images, fonts)
├── src/
│   ├── pages/
│   │   ├── index.html          ← Portal home / "classified access" landing
│   │   ├── epictome/           ← The 16-section master doc
│   │   ├── agents/             ← Character articles (Ouch, Stan, C.B., Trip)
│   │   ├── cosmology/          ← Glass, Unified Field, Seasoms
│   │   ├── terminology/        ← Glossary / definitions
│   │   ├── synchronicities/    ← The 10 Scigns log
│   │   ├── albums/             ← Music project pages
│   │   └── source-streams/     ← The 10 source streams
│   ├── data/
│   │   ├── terminology.json    ← Machine-readable glossary
│   │   ├── characters.json
│   │   └── synchronicities.json
│   └── styles/
│       └── heircor.css         ← The navy/gold/red design system
```

### The Heircor Op. Web Aesthetic (Translation of Docx to CSS)
```css
:root {
  --navy-deep:   #1A1A2E;
  --navy-mid:    #1B3A5C;
  --gold:        #C9A84C;
  --crimson:     #8B0000;
  --gray-body:   #2A2A2A;
  --gray-meta:   #666666;
  --gray-ghost:  #CCCCCC;
  --bg:          #0D0D1A;  /* darker than navy for true "screen" feel */
}

/* Section headers → full-width navy bands with white caps */
/* H2 → gold left border (gold replaces red for readability on dark bg) */
/* Intel boxes → dark panels with red left border, ghost italic text */
/* Stamps → red uppercase tracking labels */
/* Links → gold underline, no standard blue */
/* Navigation → classified file tabs / drawer */
```

### Key Design Decisions to Make
1. **Authentication feel vs. open access** — Does the portal "feel" like you're logging into a classified system (fake login screen, access granted animation) or just browse freely?
2. **Article cross-linking** — Every term in the glossary links to its full article; character names link to character pages
3. **"Intel Drop" update model** — New content published as dated "field reports" (blog-like) rather than editing existing pages
4. **Search** — Pagefind (static search for Astro) works beautifully for this
5. **Synchronicities as live log** — A running feed that grows as new Scigns are identified

### Suggested First Sprint (New Session Can Build This)
1. Design system in CSS (heircor.css with all variables, type scale, components)
2. Home / landing page (the "portal access" screen)
3. One full section rendered (Agents page — Ouch/Stan/C.B./Trip with full intel boxes)
4. Navigation structure
5. Terminology glossary page (auto-generated from JSON)

This gives a working prototype that demonstrates the full aesthetic and architecture before filling in all 16 sections.
