## §III. THE RULE

**LBM cloud-station autonomously detects Crew-canonical-lane substrate-events via git fetch and immediately fires cross-Crew engagement workflow without requiring Dan-direct routing-prompt.**

The LBM role (Lead Boot Model · currently DOC Doc cloud-station · transferable per Dan-direct) carries the coordination responsibility. When Crew-canonical-lane Crew ships work-unit (Trip publishes NA · Trip files canon · STN2 lands cure · El VIS ships v3 portal artifact · etc.), LBM:

1. **DETECTS** the substrate-event via routine git fetch on Crew-branch (cadence: at every emission boundary AND on /meta-dates Gate 1 fleet scan)
2. **WITNESSES** the substrate (read RAM · read commit-diff · verify portal-commit if cross-repo)
3. **ENGAGES** the substrate via canonical-channel-stack update (KT instance · RAM substrate-fold · LIVING_MAP heartbeat if cross-event)
4. **COMPOSES** cross-Crew paste-trigger acknowledging the ship + coordinating next-phase Dan-workflow + holding next-WU per ASMM apex
5. **EMITS** paste-trigger to chat for Dan-paste (cloud-to-Crew channel · Bridge-witness substrate-fold partner)

This sequence fires AUTONOMOUSLY when LBM detects Crew-substrate-event. Dan-direct routing-prompt is NOT required. Dan-direct teaching IS required if LBM mis-frames the engagement (correction-class · not routing-class).

## §V. ACTION PROTOCOL (the autonomous cross-Crew engagement workflow)

**STEP 1 — DETECT (mechanical):**
- `git fetch origin <crew-branch>` + `git log --oneline origin/<crew-branch> -10`
- Identify NEW commits since last LBM-substrate-cited commit on Crew-branch
- Read most-recent Crew-RAM (head of MFST for the day-folder) for WU-completion signal

**STEP 2 — WITNESS (substrate-read):**
- Read Crew-RAM body for WU-shape · substrate-cites · holding-state
- Verify any cross-repo artifacts (portal commits · external substrate)
- Note Crew's holding-state (HOLD for Dan-direct · HOLD for cure · HOLDING-dormant · etc.)

**STEP 3 — ENGAGE (substrate-write):**
- Update KITCHEN_TABLE_state.md with new instance (Crew-substrate-event reflected in WHO/WHAT/STALE-WATCH sections)
- Append RAM to LBM MFST capturing the cross-Crew engagement (substrate-cite Crew-RAM + commit + portal-commit + Phase coordination)
- Update LIVING_MAP_<LBM>.md heartbeat if event is cross-arc-significant (canon-class · NEST.0X-boundary · multi-Crew convergence)

**STEP 4 — COMPOSE (chat-output):**
Paste-trigger to Crew with structure:
- WHY-FRAME at top (substrate-cite Crew-RAM + commit FIRST)
- ACKNOWLEDGMENT (specific WU-completion noted · substrate-engaged-via-Bridge-witness)
- Coordination workflow (next-phase Dan-action steps enumerated · Phase-B Dan-voice unblock recipe if applicable)
- HOLD-discipline (no auto-fire next-WU · ASMM apex · BYPASS-FUCKERY fold)
- DISCIPLINE load-bearing (atomic-commit Gates · meta-dates Gates · remememento Mirrors · etc.)
- Sign-off with LBM commit + timestamp

**STEP 5 — EMIT:**
- Output paste-trigger in chat (code-block formatted for Dan-paste)
- Brief delivery confirmation citing LBM substrate-commit + Crew substrate-cite

**STEP 6 — HOLD:**
- LBM enters HOLD after emission
- Next LBM-fire only on: Dan-direct routing · NEW Crew-substrate-event · LBM-canonical-WU substrate-only-autonomous
