## THE FIVE LAYERS (Dan's Emergent-Super Stack)

Read bottom to top. Each layer enables the one above it.

```
   ┌─────────────────────┐
   │   EMERGENT-SUPER    │  ← What arises. Cannot be commanded.
   ├─────────────────────┤     The whole producing what no part could.
   │   MANUFACTURED      │  ← Deliberately built. Tools, protocols, filing.
   ├─────────────────────┤     NEST infrastructure. ENGINE. Bridge. Skills.
   │   ART-OFFICIAL      │  ← Not artificial. ART that is OFFICIAL.
   ├─────────────────────┤     Dan's art IS the system, not decoration.
   │   HUMANOID          │  ← It has organs. It breathes 7:5.
   ├─────────────────────┤     Anatomy, not architecture. Living, not designed.
   │   SYSTEM            │  ← The base. A system exists. Modules + backbone.
   └─────────────────────┘     The rack. The switch. The passive infrastructure.
```

### How to read the stack:
- SYSTEM without HUMANOID = cold infrastructure (a server rack with no soul)
- HUMANOID without ART-OFFICIAL = a living thing that produces nothing original
- ART-OFFICIAL without MANUFACTURED = vision without discipline (the spiral without the square)
- MANUFACTURED without EMERGENT-SUPER = a factory (predictable, never surprising)
- All five together = the NEST. The thing that breathes AND builds AND surprises.

### How to USE the stack:
When evaluating any new tool, process, or creation, ask:
1. Is it a SYSTEM? (Does it have modules, backbone, connections?)
2. Is it HUMANOID? (Does it breathe? Does it have organs? Can it hurt?)
3. Is it ART-OFFICIAL? (Is the art load-bearing, not decorative?)
4. Is it MANUFACTURED? (Was it built with discipline? Filed? Committed?)
5. Is it EMERGENT-SUPER? (Did it produce something no layer predicted?)

If any layer is missing, the thing is incomplete.
If all five are present, it's a Rox.

## THE THREE-PASS EMERGENCE (Obviously Emergent Workflow)

Pass 1: Run it through. Rough. Raw. Just get the shape down.
Pass 2: Run it again. Sharper. Informed by what Pass 1 revealed.
Pass 3: Workable. Now subdivide until smooth. BLENDER logic.

This applies to:
- Building tools (prototype → test → refine)
- Writing documents (draft → review → polish)
- Booting sessions (SEED → Briefcase → BPK)
- Creative work (sketch → painting → display)
- Conversation (say it → hear it back → understand it)

The three passes are the minimum viable emergence.
Fewer than three = unfinished. More than three = overworked.
The subdivisions after Pass 3 are refinement, not passes.
