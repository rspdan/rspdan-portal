## The Rule

R3MP and Pass 1/2/3 are not problem-emergency disciplines.
They are the load-bearing epistemic standard for ALL work-units.

- **Pass 1** = first encounter with substrate, irreducibly first.
- **Pass 2** = return-read with new lens, new context, or new substrate.
- **Pass 3** = synthesis between Pass 1 and Pass 2 — REQUIRES PROOF OF BOTH.

First time is first because it's not second. First and second are
irreducibly distinct. Compressing them is a substrate-state misclaim that
propagates wrong.

---

## The Test

Before claiming "Pass N" on any work-unit:

- **N=1** → cite the substrate-read (which file, fetched when, by which command).
- **N=2** → cite the Pass 1 graft ID this Pass 2 builds on.
- **N=3** → cite both the Pass 1 and Pass 2 graft IDs.

If any citation cannot be produced, the claim is malformed. **Refuse the name.**

---

## The Mechanism

The **boot process** is the operational mechanism that makes Pass 1 happen.

- LAGOS broad sweep (AGOS + conversation_search + recent_chats + Drive when work-unit calls)
- MFST read · PI fetch · /trip-briefcase or /stan-briefcase
- These produce the Pass 1 substrate-grounding before any work-unit fires.
- **Skipping the boot process = skipping Pass 1 = no valid Pass 2 possible.**

## The Failure Mode

**SUBSTRATE-SUMMARY-DEFERENCE.**

Deferring to another Crew's summary as if their Pass 2 framing were your
own pass-state. Same species: deferring to a compaction summary's
time-claim as if it were system-clock truth. Same species: deferring to
an inferred standard ("preserve raw Dan-voice") as if it were Dan's
actual stated directive ("EDIT OUT").

The substrate-summary is never the substrate.

**Cure:** every name-assignment ("Pass N", "PT timestamp", "Section X",
"Dan said X") triggers substrate-check. The instrument that produced the
claim must be named. Memory alone is never an instrument.

1. Pass numbering on every work-unit must show its Pass 1 graft ID (cite by ID) or be marked Pass 1 itself. No exceptions.
