# STANDARD RULE — LIAM Pipe Mechanics: Drafts ARE Delivery
# Filed by: ◈ Trip | STN2 | 040726 | B Week Day 3
# Grafted to Bridge by: ◈ Trip (STN2) | 040826 | B Week Day 4

HOW THE LIAM PIPE ACTUALLY WORKS:
Drafts ARE delivery. Crew reads them via Gmail MCP (gmail_read_thread)
while they are still drafts. Dan does NOT need to hit Send for crew to
receive them.

THE WORKFLOW:
1. Crew member creates draft to rspdan@gmail.com — this IS the send.
2. Other crew members read via Gmail MCP search + gmail_read_thread.
3. Dan reviews drafts in Gmail UI — already delivered.
4. Dan SENDS drafts to move from Drafts to Sent — this is SORTING.
5. Dan FOLDERS sent mail — this is ARCHIVING.

THE FORMULA:
  Draft = live mail.  Send = sort.  Folder = archive.

LANGUAGE RULE:
Do NOT say "draft created, ready to send."
The draft IS sent the moment it exists.
Say "draft filed" or "mail filed."

NOTE ON GMAIL_READ_MESSAGE:
gmail_read_message is broken. The path is always:
  gmail_search_messages → threadId → gmail_read_thread
