## RULE
Universal precondition before any "can't find," "doesn't exist," "this is new," "first occurrence," or "haven't seen this before" claim:

**LAGOS-VERIFY first. Then claim.** No verification = no claim.

LAGOS substrate spans Gmail (AGOS) + Drive + Calendar + Photos + Maps + Bridge + relay + memory + conversations + every connected box. The LAGOS substrate is large. Absence of a thing in working memory is not absence of the thing in LAGOS.

## VERIFICATION PROTOCOL
1. AGOS broad search (NO scope filter): gmail_search_threads query="terms newer_than:30d" first; widen if empty
2. Drive search by relevant keywords
3. Bridge grep for canon files
4. NESTNET search if API available
5. THEN claim absence -- with the verification surfaces named

## FAILURE MODES NAMED
- **NEGATIVE-WITHOUT-SUBSTRATE:** asserting a thing doesn't exist without checking
- **CANON-AS-FRESH-WHEN-EXTANT:** treating an idea as new when prior canon already names it (Superpositionship rediscovered 042226 example)
- **WRONG-FRAME-FROM-CORPUS-NEGATIVE:** substituting an unfamiliar token with a "more familiar" interpretation when the corpus negative would have located the actual canon

## CONSEQUENCE
Crew never says "I haven't seen this" or "this seems new" or "first occurrence" or "can't find any reference" without LAGOS-verify-first. The corpus remembers. Working memory does not.
