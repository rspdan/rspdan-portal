## THE RULE

Removable drives are never listed in the iCan table unless they appear
in the live drive scan.

FAS2 is a portable 1TB SSD. It mounts at L:\ when Dan plugs it in.
When it is not plugged in, it does not exist on this station.
It has no permanent place in the iCan roster.

**DO NOT:**
- List FAS2 / L:\ as a status row with "not mounted (normal)"
- Note the absence of any removable drive
- Frame a missing removable drive as an expected condition

**DO:**
- Run the drive scan (Step 0)
- Report only what the scan actually shows
- If FAS2 appears in the scan, add it to the table
- If FAS2 does not appear, say nothing about it

## THE PRINCIPLE

Naming a missing drive implies it belongs. Reporting its absence as
"normal" trains the boot to expect it. The correct posture: the drive
roster contains only what is physically present and detected.

FAS3 (D:\STAN\) is the permanent home filesystem on STN2.
Everything else is conditional on the scan.

## WHY THIS RULE EXISTS

033026 CONTACT boot: drive scan correctly detected C/D/E. iCan report
re-introduced FAS2/L:\ as a status row. Dan corrected it twice — once
in words, then with a screenshot of Explorer showing three drives and
no L:\. The second correction produced this rule.

The correction is the data. The rule is the extraction.
