## CANDIDATE STANDARD RULE — 5-REGISTER ARCHAEOLOGY

Before claiming any term, concept, or canon item is "newly coined," "clean field," or "novel," sweep all five of these registers:

1. **Recent LNL** — session notes from the past 7-14 days
2. **NOW_ANTHROPOLOGY folder on Drive** — the journal archive; many concepts get formalized here first
3. **Dan's personal Notes docs on Drive** — especially any stream-of-consciousness files (090125, etc.); seed fields often predate NEST canonicalization by months
4. **NESTNET API** — https://www.rspdan.com/api/search?q=TERM&key=NEST_API_KEY — when fetchable; falls back to Drive full-text if API blocked
5. **GraftOS with 8-hour+ window** — Gmail search `subject:[CREW] newer_than:8h` minimum for same-day handoffs; wider for multi-day continuity

If any of the five returns a hit, the field is not clean. If a term appears in Dan's pre-NEST Notes, it is not NEST-coined. If Stan, Trip, or C.B. has surfaced it previously in any register, that surfacing is upstream canon and must be cited.

**Operational rule:** "I don't know if this is new" is a better default than "this is new."
