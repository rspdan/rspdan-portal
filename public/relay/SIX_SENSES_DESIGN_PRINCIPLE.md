## APPLICATION TO OTHER TOOLS

The Senses are not file operations. They are PERCEPTUAL MODES.
Any tool can be evaluated by which senses it activates:

A camera = SEE
A journal = REMEMBER
A synthesizer = IMAGINE
A printer = BUILD
An inbox = FLOW
A debugger = TROUBLESHOOT

A tool that activates multiple senses is more alive than one that activates one.
The portal activates all six. That's why it works.

## APPLICATION TO THE DARKIVE / THEATER

The Theater (theater.py projection system) needs all six senses:
- SEE: project scanned material onto the wall (the darklight)
- REMEMBER: pull from DRK storage (the deep archive)
- IMAGINE: overlay, annotate, connect (the creative layer)
- BUILD: export to Source, commit to canon (the permanent layer)
- FLOW: intake new scans, process queue (the river)
- TROUBLESHOOT: verify, cross-check, flag inconsistencies (C.B.'s job)

## APPLICATION TO "JUNK SITTING AROUND"

Every piece of "junk" is a tool missing its sense assignment.
An old drive = a REMEMBER organ waiting for a body.
A spare monitor = a SEE organ waiting to be mounted.
An unused SSD = a FLOW organ waiting for a river.
A disconnected speaker = an IMAGINE organ waiting for Usic.

The Workshop Cube Spec assigned senses to drives.
The Six Senses principle assigns senses to EVERYTHING.

"We can make something out of all this junk sitting around."
Yes. By asking: which sense does this object serve?
Then connecting it to the system that needs that sense.
