## THE SECURITY MODEL

### Three Principles

1. **Trip reviews everything first.** Nothing moves from _INTAKE to _RELEASED without passing through _TRIP_REVIEW.
2. **Stan only sees _RELEASED.** When Trip clears material, it goes to the appropriate subfolder. Stan pulls from there. He never navigates above _RELEASED.
3. **The catalog is the audit trail.** Every item gets logged in _CATALOG.md when it enters and when its status changes.

### Clearance Levels

| Level | Meaning | Who Can See |
|-------|---------|-------------|
| INTAKE | Raw, unreviewed | Trip only |
| REVIEW | Being examined | Trip only |
| CLEARED | Safe, no personal info | Trip, then → _RELEASED |
| REDACT_FIRST | Contains real names/places/bio | Trip only until edited |
| HOLD | Safe but not yet needed | Trip only |
| RELEASED | Approved for crew use | Stan, C.B. (via Dan) |
