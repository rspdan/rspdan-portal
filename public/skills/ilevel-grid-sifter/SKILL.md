## THE THREE LAYERS

### iLevel 0 — Director Layer (load order)

What lives here: things that affect everything downstream.
If this is wrong, all downstream work is contaminated.

Action: VERIFY → FIX → CONFIRM → proceed to -1

Examples: stale seed documents, wrong temporal assumptions, hardcoded paths

### iLevel -1 — Surface Layer (operational, buildable now)

What lives here: concrete outputs Stan can build.
Some require Dan's Go; all are clearly scoped.

Action: MAP → BUILD → TEST → FILE → ADVANCE

Examples: registry files, SKILL.md writes, portal pages, /logs processing

### iLevel -2 — Substrate Layer (architecture, gates)

What lives here: things that change how the system THINKS.
Requires Dan's Go. Cannot proceed without it.

Action: IDENTIFY → SPEC → WAIT FOR GO → BUILD

Examples: nest_sleep.py, VBS launcher, YomygdylO pattern extension

## THE METHOD: LOG → MAP → SIFT → EXAMINE → NAME

**LOG:** Dump everything flat. No order, no judgment. 2-5 minutes.

**MAP:** Assign iLevel to each item using: "If this is wrong, does it contaminate others?"
→ 0. "Can I build this now?" → -1. "Does this need Go and change fundamentals?" → -2.

**SIFT:** Find dependencies within layers. Which -1 items block others? Which -2 items unblock multiple -1 items?

**EXAMINE:** Read closely. Grade each find:
- DIRT — common, process quickly
- ROCKS — notable, log and continue
- DAYCITE — obviously significant, easy to find
- NIGHTCITE — only visible through careful sifting
- QUARTZ — genuine pattern or convergence
- CERAMICS — from another stratum entirely, may reframe everything

**NAME:** When you find quartz or ceramics: STOP. File before continuing.

RULE: Strike gold → stop → file it → RELAY/OBVIOUSLY_EMERGENT_[name]_[date].md → then continue.
